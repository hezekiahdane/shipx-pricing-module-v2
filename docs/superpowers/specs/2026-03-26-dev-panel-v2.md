# Dev Panel v2 Enhancement Design

> **For agentic workers:** This spec covers three enhancements to the existing dev panel. Implement as a new branch built on top of `feature/dev-panel` (or main after it merges). Each feature is independent — implement in the order listed.

**Goal:** Make the dev panel fully automatic — pages, state simulators, and assets all self-populate based on what exists in the project and what is currently rendered on the page, with no manual config required beyond the initial setup.

**Builds on:** `src/components/dev/` — the existing DevPanel, DevPanelProvider, DevPanelContext, and section components from Phase 11.

---

## Feature 1: Pages — Auto-scan + Route Blocking

### Overview

Replace the manual `pages` array in `DevPanelConfig` with automatic filesystem discovery. Add per-page route blocking via a cookie read by the Next.js middleware.

### 1A — Auto-scan API Route

A dev-only API route at `GET /api/dev/pages` walks the `src/app/[locale]/` directory at runtime using Node's `fs` module and returns all discovered `page.tsx` routes.

**Response shape:**
```ts
{ pages: Array<{ path: string; label: string }> }
// e.g. { pages: [{ path: '/about', label: 'About' }, { path: '/contact', label: 'Contact' }] }
```

**Label derivation:** Split the path segment by `-`, capitalise each word. `/our-team` → `Our Team`. Root `/` → `Home`.

**Guard:** The route uses `withApi` from `@/lib/core/api/with-api` (required by CLAUDE.md for all API routes). Use `withApi({ csrf: false })` since this is a read-only dev-only route. Inside the handler, throw `NotFoundError` from `@/lib/core/api/errors` if `process.env.NODE_ENV !== 'development' && process.env.NEXT_PUBLIC_VERCEL_ENV !== 'preview'`. Never exposed in production.

**Config override:** Pages manually listed in `DevPanelConfig.pages` take precedence — their `label` and `status` override the auto-scanned defaults. Auto-scanned pages not in config default to `status: 'todo'`.

**`pages` field type change:** `pages` in `DevPanelConfig` changes from required to optional (`pages?: DevPanelPage[]`). When omitted, auto-scan provides all pages with `status: 'todo'`. Existing configs that list pages continue to work as overrides.

**Provider fetch:** `DevPanelProvider` fetches `/api/dev/pages` on mount via `useEffect`. Merges results with `config.pages`. Stores merged list in state as `discoveredPages`.

### 1B — Route Blocking

Each page row in `PagesSection` gets a lock icon button (🔒/🔓). Toggling it calls `setPageBlocked(path, boolean)` in context.

**Cookie:** `DevPanelProvider` writes a `devpanel_blocked` cookie whenever `blockedPages` state changes:
- Value: JSON-encoded array of blocked paths
- `SameSite=Strict; Path=/; Max-Age=86400` (expires after 24 hours)
- `DevPanelProvider` also deletes the cookie on unmount (sets `Max-Age=0`) to prevent stale blocks persisting across restarts

**Middleware:** The existing `src/middleware.ts` reads `devpanel_blocked` cookie in dev/preview only. The middleware receives the full locale-prefixed pathname from next-intl. Strip the locale prefix before matching against blocked paths using next-intl's `routing.locales` array:

```ts
// middleware.ts addition — runs only when isDev (existing guard)
const blocked: string[] = JSON.parse(cookies.get('devpanel_blocked')?.value ?? '[]');

// Strip locale prefix using routing.locales from next-intl config
const localePrefix = routing.locales.find((l) => pathname.startsWith(`/${l}/`) || pathname === `/${l}`);
const strippedPath = localePrefix
  ? pathname.slice(`/${localePrefix}`.length) || '/'
  : pathname;

if (blocked.includes(strippedPath)) {
  // Rewrite to locale-prefixed not-found so next-intl routing is respected
  const locale = localePrefix ?? routing.defaultLocale;
  return NextResponse.rewrite(new URL(`/${locale}/not-found`, request.url));
}
```

**UI:** Blocked pages show the lock icon filled + row text dimmed. Unblocking restores normal appearance.

## Feature 2: State Simulators — Auto-registration Hook

### Overview

Replace the static `config.stateSimulators` array with a dynamic registration system. Components opt in by calling a hook, which registers them into the panel on mount and unregisters on unmount. The contact form is wired as the first consumer.

### 2A — `useSimulatedState` Hook

```ts
// src/components/dev/useSimulatedState.ts
function useSimulatedState(
  id: string,
  label: string,
  states: string[],
  defaultState: string,
): string
```

- Calls `registerSimulator({ id, label, states })` from `DevPanelContext` on mount
- Calls `unregisterSimulator(id)` on unmount
- Returns `simulatedStates[id] ?? defaultState`
- Safe to call outside provider (falls back to `defaultState` via `defaultDevPanelState`)

### 2B — Context Changes

Add to `DevPanelContextValue`:
```ts
registeredSimulators: DevPanelStateSimulator[];
registerSimulator: (sim: DevPanelStateSimulator) => void;
unregisterSimulator: (id: string) => void;
```

`DevPanelProvider` manages `registeredSimulators` in state. `StateSimSection` becomes a no-prop component — it reads `registeredSimulators` and `simulatedStates` directly from context via `useDevPanel()` instead of receiving them as props.

`DevPanel.tsx` stops passing simulator props to `StateSimSection` (the `simulators`, `activeStates`, `onStateChange` props are removed from the `StateSimSection` call site).

`stateSimulators` is removed from `DevPanelConfig` — no longer needed.

### 2C — Unregistered Form Detection Hint

`DevPanelProvider` runs a deferred form scan after each pathname change: a `useEffect` on `usePathname()` that fires `setTimeout(() => scanForms(), 300)` to allow the new page's DOM to settle before querying.

The scan calls `document.querySelectorAll('form')` and counts forms whose `id` or `data-sim-id` attribute does not match any `registeredSimulators[].id`. If any unregistered forms are found, `StateSimSection` renders a hint at the bottom:

> *"N form(s) detected — add `useSimulatedState` to wire states."*

Hint is dimmed text, not an error. Zero unregistered forms = hint hidden.

### 2D — Contact Form Wiring

`src/features/contact/components/ContactForm.tsx` adds:
```ts
const simState = useSimulatedState(
  'contact-form',
  'Contact Form',
  ['idle', 'loading', 'success', 'error'],
  'idle',
);
```

Contact form reads `simState` and overrides its internal display:
- `loading` → show spinner, disable submit
- `success` → show success message
- `error` → show error message
- `idle` → normal state

Simulated state only overrides the **display** — it does not fire actual API calls.

## Feature 3: Assets — DOM Scan + Show-more

### Overview

Replace the static `config.assets` array with live DOM scanning. `AssetSection` queries the document for all media elements on the current page and renders them with a collapsible show-more UI.

### 3A — DOM Scan

`AssetSection` runs a deferred scan on mount and on pathname change (via `usePathname`). Same 300ms deferral pattern as Feature 2 form scan — allows the new page's DOM to settle:

```ts
const imgs = Array.from(document.querySelectorAll('img[src]')).map((el) => ({
  type: 'img' as const,
  // Fall back to filename when dimensions are unavailable (lazy-loaded / not yet decoded)
  label: el.alt || el.src.split('/').pop() || 'image',
  value: el.naturalWidth && el.naturalHeight
    ? `${el.naturalWidth}×${el.naturalHeight}`
    : el.src.split('/').pop() ?? '—',
  src: el.src,
}));
const videos = Array.from(document.querySelectorAll('video[src]')).map((el) => ({
  type: 'video' as const,
  label: el.getAttribute('aria-label') || el.src.split('/').pop() || 'video',
  value: el.src.split('/').pop() ?? '',
  src: el.src,
}));
const svgs = Array.from(document.querySelectorAll('svg[data-src], svg[aria-label]')).map((el) => ({
  type: 'svg' as const,
  label: el.getAttribute('aria-label') || el.getAttribute('data-src') || 'svg',
  value: '',
  src: '',
}));
```

Results are merged and stored in local state. Empty result = section renders "No assets detected."

### 3B — Show-more UI

Default: render first 3 assets. If total > 3, show a "Show N more" button below. Clicking expands the full list. A "Show less" button collapses back to 3.

Collapsed/expanded state is local to the component (no context needed).

### 3C — Config Cleanup

`DevPanelAsset` type and `assets` field are removed from `DevPanelConfig` and `index.ts`. `AssetSection` no longer accepts props — it is self-contained.

| Modify | `src/components/dev/DevPanel.tsx` — remove `assets` prop pass to `AssetSection` |

## Decisions Log

| Decision | Choice | Reason |
|----------|--------|--------|
| Page discovery | API route filesystem scan | Zero-config, auto-updates as files are created |
| API route wrapper | `withApi({ csrf: false })` | CLAUDE.md hard rule — all API routes use withApi |
| Route blocking | Cookie + middleware rewrite | Works on hard navigation, no server restart, dev-only |
| Cookie expiry | 24h + cleanup on unmount | Prevents stale blocks persisting into production or after restarts |
| Middleware locale strip | `routing.locales` array from next-intl config | Reuses existing routing config, no new utility needed |
| Rewrite target | `/${locale}/not-found` | Respects next-intl locale routing, avoids redirect loops |
| `pages` field | Optional (`pages?: DevPanelPage[]`) | Auto-scan replaces manual list; existing configs still work as overrides |
| State sim discovery | `useSimulatedState` opt-in hook | Components must cooperate to override internal state |
| Form scan timing | 300ms deferred `setTimeout` | DOM settles after pathname change before querying |
| Unregistered form nudge | Hint text (not error) | Informs without blocking |
| `StateSimSection` props | Removed — reads context directly | Consistent with no-prop auto pattern; `DevPanel.tsx` updated accordingly |
| Asset discovery | DOM scan on mount + pathname change (300ms deferred) | Fully automatic, no config required |
| Image dimension fallback | Filename when `naturalWidth === 0` | Handles lazy-loaded / not-yet-decoded images gracefully |
| Assets config | Removed entirely | DOM scan replaces it |
| State sim config | Removed entirely | Auto-registration replaces it |
